Java OOP – Lớp (Class) và Đối tượng (Object)

🗓️ 28/10/2025

🔰 1. Tổng quan

Trong lập trình hướng đối tượng (OOP), hai khái niệm cơ bản nhất chính là Lớp (Class) và Đối tượng (Object).
Chúng giúp mô hình hóa các thực thể trong thế giới thực dưới dạng lập trình.

  • Class: là bản thiết kế (blueprint) hay khuôn mẫu mô tả kiểu dữ liệu, đặc tính, hành vi của một đối tượng.
  • Object: là thực thể cụ thể được tạo ra từ một Class. Mỗi object có trạng thái (state) và hành vi (behavior) riêng.

🧱 2. Lớp (Class) trong Java

Một Class trong Java có thể chứa:

  • Trường (fields) → lưu trữ dữ liệu (trạng thái)
  • Hàm tạo (constructor) → khởi tạo đối tượng
  • Phương thức (methods) → mô tả hành vi

Ví dụ, ta tạo một lớp mô tả ô tô (Car):

class Car {

    // Thuộc tính (fields)
    String type;
    String model;
    String color;
    int speed;

    // Hàm tạo (constructor)
    Car(String type, String model, String color) {
        this.type = type;
        this.model = model;
        this.color = color;
    }
    
    // Phương thức (method)
    int increaseSpeed(int increment) {
        this.speed = this.speed + increment;
        return this.speed;
    }
}

Lớp trên định nghĩa cách mô tả một chiếc xe hơi. Khi ta tạo đối tượng từ class, ta đang tạo ra một “phiên bản cụ thể” của chiếc xe đó.

⚙️ Hàm tạo mặc định (Default Constructor)

Mỗi lớp Java đều có một constructor mặc định nếu ta không tự định nghĩa:

Car() { }

Constructor mặc định này gán các giá trị mặc định (chuỗi → null, số → 0).

Nếu ta muốn các đối tượng có giá trị khởi tạo riêng, ta định nghĩa constructor như sau:

Car(String type, String model) {
    this.type = type;
    this.model = model;
}

3. Đối tượng (Object)

Khi chương trình chạy (runtime), ta tạo đối tượng từ class bằng từ khóa new:

Car focus = new Car("Ford", "Focus", "red");
Car auris = new Car("Toyota", "Auris", "blue");
Car golf = new Car("Volkswagen", "Golf", "green");

Mỗi dòng trên tạo ra một object (xe hơi riêng biệt) dựa trên cùng một bản thiết kế Car.

Ta có thể thay đổi trạng thái (ví dụ: tốc độ) của từng đối tượng qua phương thức:

focus.increaseSpeed(10);
auris.increaseSpeed(20);
golf.increaseSpeed(30);

➡️ Giờ đây, ba chiếc xe có tốc độ khác nhau – minh họa cách mỗi object có trạng thái riêng biệt.

4. Phạm vi truy cập (Access Modifiers)

Trong ví dụ trên, ta chưa định nghĩa phạm vi truy cập cho class, field hay method.

Các modifier phổ biến gồm:

Modifier Mức truy cập Ghi chú
public Truy cập từ mọi nơi Thường dùng cho class, method
private Chỉ truy cập trong class Dùng để ẩn dữ liệu nội bộ
protected Truy cập trong package và các lớp con Dùng trong kế thừa
(default) Truy cập trong cùng package Khi không ghi rõ modifier

Ví dụ class hoàn chỉnh với access modifier:

public class Car {
    private String type;
    private String model;
    private String color;
    private int speed;

    // Constructor công khai
    public Car(String type, String model, String color) {
        this.type = type;
        this.model = model;
        this.color = color;
    }

    // Getter & Setter
    public String getColor() {
        return color;
    }

    public void setColor(String color) {
        this.color = color;
    }

    public int getSpeed() {
        return speed;
    }

    // Phương thức tăng tốc
    public void increaseSpeed(int increment) {
        this.speed += increment;
    }
}

➡️ Ở đây:

  • Lớp và constructor là public → có thể tạo object từ mọi nơi.

  • Các field là private → chỉ truy cập thông qua getter/setter.

  • Đây chính là ví dụ của tính đóng gói (Encapsulation) trong OOP.

🧩 5. Kết luận

Trong bài này, ta đã tìm hiểu:

  • Class là khuôn mẫu mô tả dữ liệu & hành vi.

  • Object là thể hiện cụ thể được tạo từ class.

  • Access Modifier giúp bảo vệ và kiểm soát truy cập dữ liệu nội bộ.

💡 Bước tiếp theo: Tìm hiểu về Kế thừa (Inheritance), Từ khóa super, và Lớp trừu tượng (Abstract class) — những nền tảng tiếp theo của OOP trong Java.

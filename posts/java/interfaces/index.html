Interface trong Java – Đa hình và Kế thừa đa cấp

🗓️ 29/10/2025

1. Tổng quan

Trong bài viết này, chúng ta sẽ cùng tìm hiểu Interface trong Java, cách mà Java sử dụng chúng để thực hiện đa hình (polymorphism) và kế thừa đa cấp (multiple inheritance).

2. Interface trong Java là gì?

Trong Java, interface là một kiểu trừu tượng (abstract type) chứa tập hợp các phương thức (methods) và hằng số (constants).
Interface là một trong những khái niệm cốt lõi giúp Java đạt được tính trừu tượng, đa hình và kế thừa đa cấp.

Ví dụ đơn giản:

public interface Electronic {

    // Biến hằng
    String LED = "LED";

    // Phương thức trừu tượng
    int getElectricityUse();

    // Phương thức static
    static boolean isEnergyEfficient(String electronicType) {
        return electronicType.equals(LED);
    }

    // Phương thức mặc định (default)
    default void printDescription() {
        System.out.println("Electronic Description");
    }
}

Chúng ta có thể triển khai interface này trong một lớp bằng từ khóa implements:

public class Computer implements Electronic {

    @Override
    public int getElectricityUse() {
        return 1000;
    }
}

2.1. Quy tắc khi tạo Interface

Trong interface, chúng ta có thể sử dụng:

  • Biến hằng (constant variables)

  • Phương thức trừu tượng (abstract methods)

  • Phương thức tĩnh (static methods)

  • Phương thức mặc định (default methods)

Một số quy tắc quan trọng:

  • Không thể khởi tạo interface trực tiếp.

  • Interface có thể rỗng, không chứa phương thức nào.

  • Không thể dùng từ khóa final cho interface.

  • Phương thức trong interface mặc định là public abstract.

  • Biến trong interface luôn là public static final.

  • Từ Java 9 trở đi, có thể định nghĩa phương thức private trong interface.

3. Lợi ích của Interface

31. Cung cấp hành vi chung (Behavioral Functionality)

Interface giúp chúng ta thêm hành vi có thể dùng lại giữa các lớp không liên quan. Ví dụ: Comparable, Comparator, và Cloneable là các interface phổ biến trong Java.

public class Employee {
    private double salary;

    public double getSalary() { return salary; }
    public void setSalary(double salary) { this.salary = salary; }
}

public class EmployeeSalaryComparator implements Comparator<Employee> {

    @Override
    public int compare(Employee a, Employee b) {
        return Double.compare(a.getSalary(), b.getSalary());
    }
}

3.2. Kế thừa đa cấp (Multiple Inheritance)

Java không cho phép kế thừa nhiều lớp, nhưng có thể kế thừa nhiều interface.

public interface Transform {
    void transform();
}

public interface Fly {
    void fly();
}

public class Car implements Fly, Transform {

    @Override
    public void fly() {
        System.out.println("I can Fly!!");
    }

    @Override
    public void transform() {
        System.out.println("I can Transform!!");
    }
}

Ở ví dụ trên, Car kế thừa hai interface và thực thi cả hai phương thức fly() và transform().

3.3. Đa hình (Polymorphism)

Đa hình cho phép đối tượng có thể thể hiện dưới nhiều hình dạng khác nhau tại runtime. Trong Java, interface là công cụ mạnh để thực hiện điều này.

public interface Shape {
    String name();
}

public class Circle implements Shape {
    @Override
    public String name() { return "Circle"; }
}

public class Square implements Shape {
    @Override
    public String name() { return "Square"; }
}

List<Shape> shapes = new ArrayList<>();
shapes.add(new Circle());
shapes.add(new Square());

for (Shape shape : shapes) {
    System.out.println(shape.name());
}

Kết quả:

Circle
Square

4. Phương thức mặc định trong Interface

Trước Java 8, nếu bạn thêm một phương thức mới vào interface thì tất cả các lớp cũ phải override nó — gây lỗi tương thích ngược.

Từ Java 8, có thể thêm default method để tránh điều này:

public interface Printer {
    void print();
    default void info() {
        System.out.println("Default Printer Info");
    }
}

5. Quy tắc kế thừa Interface

5.1. Interface kế thừa Interface khác

Một interface có thể mở rộng (extends) interface khác:

public interface HasColor {
    String getColor();
}

public interface Box extends HasColor {
    int getHeight();
}

→ Box kế thừa HasColor, nên có cả hai phương thức getColor() và getHeight().

5.2. Abstract Class triển khai Interface

Khi một abstract class triển khai interface, nó kế thừa toàn bộ phương thức trừu tượng và mặc định của interface đó.

public interface Transform {
    void transform();
    default void printSpecs() {
        System.out.println("Transform Specification");
    }
}

public abstract class Vehicle implements Transform {}

Vehicle kế thừa cả phương thức transform() (abstract) và printSpecs() (default).

6. Functional Interface

Functional Interface là interface chỉ có một phương thức trừu tượng duy nhất. Từ Java 8, các functional interface được sử dụng phổ biến với lambda expressions.

Một số ví dụ phổ biến:

  • Runnable

  • Callable

  • Predicate

  • Consumer

  • Function<T, R>

Ví dụ:

@FunctionalInterface
public interface Greeting {
    void sayHello(String name);
}

Greeting greet = name -> System.out.println("Xin chào, " + name);
greet.sayHello("Java!");

7. Kết luận

Qua bài viết này, chúng ta đã hiểu:

  • Interface là công cụ để đạt được tính trừu tượng, đa hình và kế thừa đa cấp
  • Có thể chứa phương thức abstract, static, default
  • Giúp viết code linh hoạt, dễ mở rộng, dễ bảo trì
